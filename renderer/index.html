Post-Flash Config Modes

        
      

      

        
BurnTag supports two ways to provision configuration onto a device after flashing. Pick the one that matches how your firmware reads settings.


        
JSON over Serial

        
After the chip resets, BurnTag reopens the USB serial port and pings the firmware with a short JSON line. Once the firmware replies, BurnTag sends one config command per item and validates the response. Use this when settings change per-burn and the firmware already has a command parser.

        
Firmware contract — your sketch must:

        
    
          
  1. Print or accept a ping request and reply with JSON that matches Ready Response, e.g. {"ready":true}
    2. 
          
  2. Accept each command rendered from Command Template (with {{key}} / {{value}} interpolated) and reply with JSON that matches Expected Response, e.g. {"success":true}
    3. 
        

        
Arduino sketch sketch (ESP32-C3/S3 with native USB CDC):

        
Serial.begin(115200);
// In loop():
if (Serial.available()) {
  String line = Serial.readStringUntil('\n');
  if (line.indexOf("\"ping\"") >= 0) {
    Serial.println("{\"ready\":true}");
  } else if (/* parse and apply */) {
    Serial.println("{\"success\":true}");
  }
}

        
The old fixed "Boot Delay" field is gone — the ping handshake replaces it, which is both faster when firmware boots quickly and more reliable when it doesn't.


        
NVS Partition

        
BurnTag generates an ESP-IDF NVS partition image in memory and flashes it alongside the firmware. The device reads values on boot using the standard Preferences / nvs_get_* APIs — no serial RPC, no boot-up race. Prefer this for factory provisioning data (ID, customer key, region, model).

        
Firmware contract — your sketch reads values from the NVS namespace you configure here:

        
Preferences prefs;
prefs.begin("config", /*readOnly=*/true);
uint32_t id = prefs.getUInt("id", 0);
String key = prefs.getString("key", "");
prefs.end();

        
Partition layout — Offset and Size must match the nvs entry in your device's partitions.csv. The Arduino default is:

        
nvs,     data, nvs,     0x9000,  0x6000,

        
Each item has a type: u8 / u16 / u32 / i32 / string. If you leave the type blank, BurnTag infers it from the value (integers become u32, text becomes string). Max 15 chars for keys; max ~125 items total.


        
Writing the serial number to the device

        
Enable Serial Numbering and tick Write serial to device to bake the generated serial into the chip, not just onto the label. It piggy-backs on whichever mode is selected above (JSON or NVS), using the Device Key you set there. Choose string if your firmware reads the full "FC-000001" form, or u32/u16 if you only want the numeric part (leave the prefix blank in that case). The counter only advances after a successful burn.


        
Common

        
In both modes, Auto Increment bumps a numeric value by 1 after each successful burn. Use {config:key} in the Label Designer body lines to print any config value on the label — e.g. S/N: {config:id}.
